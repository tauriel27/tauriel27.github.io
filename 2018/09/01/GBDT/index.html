Regression Tree and Ensemble

The tree ensemble model consists of a set of classification and regression trees (CART).

  • CART

We classify the members of a family into different leaves, and assign them the score on the corresponding leaf. A CART is a bit different from decision trees, in which the leaf only contains decision values.

regression tree is a function that maps the attributes to the score

  • Regression Tree Ensemble

Usually, a single tree is not strong enough to be used in practice. What is actually used is the ensemble model, which sums the prediction of multiple trees together.

Mathematically, we can write our model in the form.where $K$
is the number of trees, $f$ is a function in the functional space $\mathcal{F}$, and $\mathcal{F}$ is the set of all possible CARTs.

$$\hat{y}i = \sum{k=1}^K f_k(x_i), f_k \in \mathcal{F}$$

  • How can we learn functions?

    • I want to predict whether I like romantic music at time t

    • Things we need to learn

    • Objective for single variable regression tree(step functions)

      • Training Loss: How will the function fit on the points?
      • Regularization: How do we define complexity of the function?
        • Number of splitting points, l2 norm of the height in each segment?
      • Define objective (loss, regularization), and optimize it!!

      The objective function to be optimized is given by

      $$\text{obj}(\theta) = \sum_i^n l(y_i, \hat{y}i) + \sum{k=1}^K \Omega(f_k)$$
      where $L$
      is the training loss function, and $Ω$
      is the regularization term.

    • Bias-variance tradeoff

Regression tree ensemble defines how you make the prediction score, it can be used for Classification, Regression, Ranking….It all depends on how you define the objective function!

Boosting Tree

  • Boosting

Boosting 是一族可将弱学习器提升为强学习器的算法。先从初始训练集训练出一个基学习器，然后根据基学习器的表现对训练样本分布进行调整，使得先前基学习器做错的训练样本受到更多的关注，然后基于调整后的样本分布来训练下一个基学习器，如此重复，直至基学习器数目达到指定值T，最终将这T个基学习器进行加权结合。

  • AdaBoost (Adaptive Boosting)

基学习器的线性组合（additive model）

$$H(x)=\sum_{t=1}^T \alpha_th_t(x)$$

来最小化指数损失函数（exponential loss function）

$$\ell_{exp}(H\rvert D)=\mathbb{E}_{x\thicksim\mathcal{D}}[e^{-f(x)H(x)}]$$

算法流程如下

  • AdaBoost算法解释

    AdaBoost算法是模型为加法模型、损失函数为指数函数、学习算法为向前分步算法时的二分类学习方法

    • 加法模型（additive model）

      $$f(x)=\sum_{m=1}^M\beta_mb(x;\gamma_m)$$
      其中，$b(x;\gamma_m)$为基函数，$\gamma_m$为基函数的参数，$\beta_m$为基函数的系数，显然AdaBoost最终分类器线性组合 $f(x)=\sum_{m=1}^M\alpha_mG_m(x)$ 是一个加法模型。

    • 向前分步加法模型

    • 问题的分解


      前向分步算法将同时求解从 $m$=1 到 $M$ 所有参数的优化问题，简化为逐次求解各个参数的优化问题。

  • AdaBoost算法推导

    • 使用指数损失函数

    • 前向分步算法

      $$f(x_i)=f_{m-1}(x_i)+\beta_mG(x_i)$$
      带入loss function

      $$w_i^{(m)}=exp(-y_if_{m-1}(x_i))$$

      $w_i^{(m)}$ 跟 $\beta$ 和 $G$都不相关，可以看作迭代中添加的一个系数，和 $f_{m-1}(x_i)$ 相关，所以该系数随着迭代会改变

      现证明上式中达到最小 $\beta^_m G^_m$ 的就是AdaBoost算法中的$\beta_m G_m$，当 $\beta>0$


      此分类器是使 $m$ 轮加权训练数据分类误差率最小的基本分类器


      将求得的 $G^*_m(x)$ 代入上式，对 $\beta$ 求导并使倒数为0，即可得到最小的 $\beta$

      $err_m$是分类误差率

      再看权值更新

      可以看出与AdaBoost各步骤等价

  • Boosting Tree 总结

    提升树实际采用加法模型与前向分步算法，以决策树为基函数的提升方法。对分类问题决策树是二叉分类树，对回归问题是二叉回归树

Gradient Boosting Decision Tree（MART）

AdaBoost通过提高错分数据点的权重来改进模型，而梯度提升树通过计算负梯度来改进模型。回归问题的提升树通过拟合残差学习回归树，这个残差是采用Squared error损失函数的负梯度，是梯度提升树的一种情况。

XGBoost

  • 二阶泰勒展开+添加正则项
  • 特征划分
    • Weighted Quantile Sketch（for approximate algorithm）
    • Sparsity-aware Split Finding（run faster for sparse data）
  • 并行加速

二阶泰勒展开

$$\hat{y}_i^{(0)} = 0 \
\hat{y}_i^{(1)} = f_1(x_i) = \hat{y}_i^{(0)} + f_1(x_i)\
\hat{y}_i^{(2)} = f_1(x_i) + f_2(x_i)= \hat{y}_i^{(1)} + f_2(x_i)\
\dots\
\hat{y}i^{(t)} = \sum{k=1}^t f_k(x_i)= \hat{y}_i^{(t-1)} + f_t(x_i)$$

  • 目标函数

$$\text{obj}^{(t)} = \sum_{i=1}^n l(y_i, \hat{y}i^{(t)}) + \sum{i=1}^t\Omega(f_i) \
= \sum_{i=1}^n l(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)) + \Omega(f_t) + \mathrm{constant}$$

  • MSE损失函数

$$\text{obj}^{(t)} = \sum_{i=1}^n (y_i - (\hat{y}_i^{(t-1)} + f_t(x_i)))^2 + \sum_{i=1}^t\Omega(f_i) \
= \sum_{i=1}^n [2(\hat{y}_i^{(t-1)} - y_i)f_t(x_i) + f_t(x_i)^2] + \Omega(f_t) + \mathrm{constant}$$
$$y_i - (\hat{y}_i^{(t-1)} + f_t(x_i))= r-f_t(x_i) \
r=y_i-\hat{y}^{t-1}$$

$r$ 一般称为残差，也即是一阶导，负梯度

使用其他损失函数无法得到这么规则的形式，考虑一般情况，对 obj 做二阶泰勒展开

$$\text{obj}^{(t)} = \sum_{i=1}^n [l(y_i, \hat{y}_i^{(t-1)}) + g_i f_t(x_i) + \frac{1}{2} h_i f_t^2(x_i)] + \Omega(f_t) + \mathrm{constant}$$

$g_i$, $h_i$ 分别为一阶导和二阶导

$$g_i = \partial_{\hat{y}_i^{(t-1)}} l(y_i, \hat{y}_i^{(t-1)})\
h_i = \partial_{\hat{y}_i^{(t-1)}}^2 l(y_i, \hat{y}_i^{(t-1)})$$

移去所有常数，在训练第 $t$ 轮时

$$\sum_{i=1}^n [g_i f_t(x_i) + \frac{1}{2} h_i f_t^2(x_i)] + \Omega(f_t)$$
得到这个一般形式，只依赖于 $g_i$ 和 $h_i$，使得XGBoost可以使用定制的损失函数

正则项

  • 树的定义

$$f_t(x) = w_{q(x)}, w \in R^T, q:R^d\rightarrow {1,2,\cdots,T} $$

w 是每个叶子的值组成的向量，q 是 x 到树叶子的映射函数，T 是叶子个数，所以模型复杂度定义如下

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \sum_{j=1}^T w_j^2$$

Shrinkage and Column Subsampling

除了添加正则化项，还使用两种额外的技术来进一步阻止overfitting。

  • 第一种技术是Friedman介绍的Shrinkage。Shrinkage会在每一步tree boosting时，会将新加入的weights通过一个因子η
    进行缩放。与随机优化中的learning rate相类似，对于用于提升模型的新增树(future trees)，shrinkage可以减少每棵单独的树、以及叶子空间（leaves space）的影响。
  • 第二个技术是列特征子抽样(column feature subsampling)。该技术也会在RandomForest中使用，在商业软件TreeNet中的gradient boosting也有实现，但开源包中没实现。根据用户的反馈，比起传统的行子抽样（row sub-sampling：同样也支持），使用列子抽样可以阻止overfitting。列子抽样的使用可以加速并行算法的计算

Structure score

$$\text{obj}^{(t)} \approx \sum_{i=1}^n [g_i w_{q(x_i)} + \frac{1}{2} h_i w_{q(x_i)}^2] + \gamma T + \frac{1}{2}\lambda \sum_{j=1}^T w_j^2\
= \sum^T_{j=1} [(\sum_{i\in I_j} g_i) w_j + \frac{1}{2} (\sum_{i\in I_j} h_i + \lambda) w_j^2 ] + \gamma T$$

$I_j = {i|q(x_i)=j}$ 是分配到编号 $j$ 叶子的所有数据点的集合

$$\text{obj}^{(t)} = \sum^T_{j=1} [G_jw_j + \frac{1}{2} (H_j+\lambda) w_j^2] +\gamma T$$

$G_j = \sum_{i\in I_j} g_i$ 表示叶子编号为j的数据点的一阶导的和， $H_j = \sum_{i\in I_j} h_i$ 表示叶子编号为j的数据点的二阶导的和

$wj$ 相互独立，$G_jw_j+\frac{1}{2}(H_j+\lambda)w_j^2$ 是个二次函数，我们可以直接求出最优点

$$w_j^\ast = -\frac{G_j}{H_j+\lambda}\
\text{obj}^\ast = -\frac{1}{2} \sum_{j=1}^T \frac{G_j^2}{H_j+\lambda} + \gamma T$$

通过计算obj可以衡量树结构的好坏，相比决策树更加精细（只考虑提升信息增益）

寻找最优划分点

Exact Greedy Algorithm（预先排序+枚举）

我们虽然可以通过obj衡量一颗树的质量，但是暴力枚举依旧困难，先考虑叶子结点的划分

$$Gain = \frac{1}{2} \left[\frac{G_L^2}{H_L+\lambda}+\frac{G_R^2}{H_R+\lambda}-\frac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$$
该公式组成：

  • 划分左叶子的增益
  • 划分右叶子的增益
  • 父节点的增益

可以看到如果划分增益小于 $\gamma$ ，那最好不要这个分支，这正是树剪枝的一个基本策略

Approximate Algorithm

数据可能没有办法全部载入内存，无法使用上述贪心策略。该算法会首先根据特征分布的百分位数(percentiles of feature distribution)，提出候选划分点(candidate splitting points)。接着，该算法将连续型特征映射到由这些候选点划分的分桶(buckets)中，聚合统计信息，基于该聚合统计找到在建议（proposal）间的最优解。

  • 寻找 $S_k$

$D_k={(x_{1k},h_1),(x_{2k},h_2)···(x_{nk},h_n)}$ 表示样本第 k 个特征值和二阶导

表示第 k 个特征值小于 z的样本比例，通过下式寻找 $D_k$, $\epsilon$ 是近似因子，可以理解为分为 $1/\epsilon$ 个划分点

  • global or local

该算法有两个变种，取决于给定的建议（proposal）。全局变种（global variant）会在树构建的初始阶段，建议所有的候选划分，并在所有的层级（level）上使用相同的建议。局部变种（local variant）则在每次划分后重新建议（re-proposes）。比起局部法，全局法需要更少的建议步骤。然而，对于全局建议，通常需要更多的候选点，因为在每次划分之后，不需要重新定义候选。局部建议会在每次划分后重新定义候选，对于更深的树更合适。图3展示了在Higgs boson数据集上不同算法的比较。我们发现，局部建议确实需要更少的候选。如果两者的候选一样多，全局建议比局部建议会更精确。

Sparsity-aware Split Finding

有些数据集本身或者由于特征工程（one-hot）导致特征非常稀疏。XGBoost提出一种针对稀疏特征的统一处理方式，同时计算开销减小（只计算非空条目）

  • 只计算非空值
  • 分别处理将missing该特征值的样本分配到左叶子结点和右叶子结点的两种情形，分到那个子节点带来的增益大，默认的方向就是哪个子节点
